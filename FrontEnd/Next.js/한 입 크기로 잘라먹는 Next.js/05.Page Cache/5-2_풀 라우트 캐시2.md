# `<suspense>`

- 빌드 타임에는 쿼라스트링, params 값 등의 동적인 값이 존재할 수 없기 때문에 해당 데이터를 받아와 사용하는 컴포넌트를 실행하면 오류가 발생하게 된다.
- 때문에 이런 동적 데이터를 사용하는 컴포넌트를 클라이언트 측에서만 실행할 수 있도록 `<suspense>`는 사전 렌더링 과정 중에서 완전히 배제하도록 설정해주는 컴포넌트이다.
- `fallback`으로 대체 UI를 설정할 수 있다.
- Next.js 서버에서 사전 렌더링을 진행할 때 `<suspense>`로 묶여 있는 컴포넌트들은 미완성 상태로 남겨저 곧바로 렌더링하지 않게되고 `fallback`으로 설정한 대체 UI가 대신 렌더링된다.
- 감싸진 컴포넌트의 비동기 작업이 종료가 될 때까지 미완성 상태로 남는다.

```js
<div>
  <Suspense fallback={<div>Loading...</div>}>
    <Searchbar />
  </Suspense>
  {children}
</div>
```
